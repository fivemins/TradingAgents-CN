# TradingAgents-CN v0.1.0

## Release Title

`TradingAgents-CN v0.1.0 - First Public Release`

## Release Summary

`TradingAgents-CN v0.1.0` is the first public release of this A-share focused fork based on the original `TradingAgents` project.  
This version packages the current local dashboard workflow into a shareable baseline for installation, evaluation, and secondary development.

It includes:

- Chinese dashboard UI for single-user local deployment
- A-share single-stock analysis
- Overnight scanner with formal picks, watchlist, and preliminary screening
- Next-open validation and 60-trading-day historical review
- Structured decision outputs and structured factor snapshots
- Windows-friendly startup and deployment flow
- QVeris supplemental routing for selected live overnight-scan data paths

## Highlights

### 1. Chinese dashboard workflow

The project now includes a Chinese-first dashboard for local use, covering:

- dashboard home
- single-stock analysis
- overnight recommendations
- task center
- task detail with structured outputs

### 2. A-share overnight research workflow

The overnight module has been extended beyond a simple pick list and now supports:

- preliminary screening
- total-score candidate review
- rejection-reason visibility
- formal recommendations and watchlist separation
- next-open validation
- historical review over recent trading days

### 3. Structured decision outputs

Single-stock tasks now produce more inspectable outputs, including:

- structured decision summaries
- factor snapshots
- evidence snapshots
- stronger UI compatibility for old and partial task records

### 4. Public-release repository cleanup

This release also prepares the repository for public collaboration by:

- removing local runtime artifacts and caches from version control
- adding deployment guides and attribution files
- improving `.gitignore`
- restoring the tracked dashboard frontend source tree

## Recommended Audience

This release is suitable for:

- friends or collaborators who want to deploy the dashboard locally
- users interested in A-share research workflows
- developers who want to continue building on top of `TradingAgents`

## Known Limitations

- Live data reliability still depends on upstream providers such as Akshare and optional QVeris routes
- The project is oriented toward research workflows rather than production trading execution
- Historical validation may still carry data-source limitations depending on cache and snapshot availability
